# Model Type - ユーザーガイド

製品の「実績スペック」を可視化するアプリケーションの使い方ガイドです。

---

##  ページ構成

### ① ウェルカムページ `/`

アプリのトップページです。ログインしていない場合はここに表示されます。

---

### ② ポータルダッシュボード `/dashboard`

**あなた専用の管理画面です。**

![マイページのイメージ]

| できること | 説明 |
|:---|:---|
| 🔍 **製品を検索** | 自分が登録した製品を名前や型番で探せます |
| 📊 **統計を確認** | 平均寿命、年間維持費、故障率などを一目で確認 |
| 📋 **詳細分析** | 製品を選ぶと、その製品の詳しい情報が表示されます |

**こんな人向け：** 自分が持っている製品を管理したい人

---

### ③ パブリックダッシュボード `/public`

**みんなのデータを集めた「集合知」の分析画面です。**

| できること | 説明 |
|:---|:---|
|  **型番で検索** | 気になる製品の型番や名前で検索 |
|  **信頼性を確認** | その製品がどれくらい壊れにくいかをスコアで表示 |
|  **コスパを確認** | 1日あたりのコスト（CPD）で経済性を評価 |
|  **製品を比較** | 最大3つの製品を並べて比較できます |

**こんな人向け：** 購入前に製品の評判を数字で知りたい人

---

### ④ 製品登録 `/products/create`

**新しい製品を登録する画面です。**

#### ステップ1: 製品を探す
- 型番や製品名を入力すると、候補が表示されます
- 楽天のデータベースからも自動でサジェストされます

#### ステップ2: 詳細を入力
- 購入日、価格、状態などを入力します
- 保証期限やメモも記録できます

**入力する主な情報：**
- 製品名・型番・メーカー
- 購入日・購入価格
- 購入状態（新品 / 中古 / 再生品）
- 現在のステータス（使用中 / 保管中 / 修理中 / 廃棄済み）

---

### ⑤ インシデント登録 `/incidents/create`

**製品に起きた問題（故障・メンテナンス）を記録する画面です。**

#### ステップ0: 製品を選ぶ
登録済みの製品から選択します

#### ステップ1: 基本情報
- いつ起きたか（発生日）
- どんな問題か（故障 / メンテナンス / 破損 / 紛失）
- どれくらい深刻か（軽微〜致命的）

#### ステップ2: 詳細情報
製品のカテゴリに応じた質問に答えます
（スマホなら画面の状態、PCならストレージの状態など）

#### ステップ3: 対応と費用
- どう対応したか（修理 / 交換 / 自己解決）
- いくらかかったか（費用）

---

### ⑥ インシデント一覧 `/incidents`

登録したインシデントの一覧を確認できます。

---

### ⑦ インシデント編集 `/incidents/{id}/edit`

登録済みのインシデントを修正する画面です。

---

##  見られる統計データ

### 製品ごとの分析（パブリックダッシュボード）

| 指標 | 意味 |
|:---|:---|
| **信頼性スコア** | 壊れにくさ（100点満点、高いほど良い） |
| **CPD** | 1日あたりのコスト（低いほどお得） |
| **平均寿命** | 同型番製品の平均使用年数 |
| **1年故障率** | 購入後1年以内に故障した割合 |
| **サンプル数** | 何台分のデータがあるか（多いほど信頼性UP） |

### グラフで見られるもの

| グラフ | 内容 |
|:---|:---|
| **レーダーチャート** | 信頼性・コスパ・寿命・修理費用を全体平均と比較 |
| **インシデント分布** | 故障/メンテ/破損/紛失の割合 |
| **発生時期パターン** | いつ頃問題が起きやすいか（0-3ヶ月、1-2年など） |
| **寿命分布** | 製品がどれくらい使われているか |
| **コスト推移** | 時期別のメンテナンス費用 |

## こんな使い方ができます

### 使い方1: 製品を買う前に調べる

1. パブリックダッシュボード (`/public`) を開く
2. 気になる製品の型番や名前で検索
3. 信頼性スコアやCPDを確認
4. 他の候補と比較して決める

### 使い方2: 自分の製品を管理する

1. 製品登録 (`/products/create`) で持っている製品を登録
2. 故障やメンテがあればインシデント登録
3. ポータルダッシュボード (`/dashboard`) で管理状況を確認

### 使い方3: データを貢献する

あなたが登録した製品・インシデントの情報は、匿名化されてパブリックダッシュボードの統計に反映されます。使えば使うほど、みんなの役に立つデータが蓄積されていきます。

##  ? よくある質問

### Q: 自分のデータは他の人に見られますか？

**A:** 個人を特定できる情報は公開されません。パブリックダッシュボードでは、型番ごとに集計された統計データのみが表示されます。

### Q: 中古で買った製品も登録できますか？

**A:** はい、登録できます。ただし、寿命の統計計算では「新品」で購入した製品のみが対象になります（より正確な数値を出すため）。

### Q: CPDって何ですか？

**A:** **Cost Per Day（1日あたりのコスト）** の略です。

```
CPD = (購入価格 + 修理費用の合計) ÷ 使用日数
```

例えば10万円で買って2年使った製品なら、CPD = 約137円/日 です。

### Q: サンプル数が少ないとどうなりますか？

**A:** サンプル数が少ないと統計の信頼性が下がります。目安として10台以上のデータがあると参考になります。
