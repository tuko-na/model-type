# 型番製品管理

個人またはグループが所有する製品を「型番（model_number）」をキーとして管理するWebアプリケーションです。製品に紐づく「インシデント（故障、メンテ履歴）」を記録・管理し、そのデータを「統計」として可視化します。

## Background

### 開発のきっかけ
既存のレビューサイトは参考になりますが、そこにあるのは「個人の主観」による言葉が多く、人によって評価基準が違ったり、曖昧だったりします。私が知りたいのは、「自分が求めている要件（性能、コスト、耐久性など）を、その製品が本当に満たしているかどうか」であるために今回制作を始めました。

### 「実績スペック」の可視化
CPUや重さなどの「カタログスペック」は公表されています。しかし、「実際どれくらい持つのか」「維持費はいくらかかるのか」といった運用後の「実績スペック」は、どこにも数値化されていません。

Model Type は、自分の持っている製品を「型番」で記録し、実績をデータ化するアプリです。「言葉」によるレビューではなく、使用期間やコストといった「数字」を積み上げてグラフ化することで、主観のズレを排除します。

カタログスペックに加えて、寿命やコストといった「実績データ」も比較項目に加えることで、この製品は自分の要件（基準）を満たしているか？を客観的に判断できるようにするために開発しました。

## Target Audience

**コアターゲット:** 20代後半〜40代の男性  
**職業:** エンジニア、理系職種、テクニカルライターなど

## Implementation Highlights (実装のこだわり)

本アプリケーションは、信頼性の高い「実績スペック」を可視化するため、データの「入力」と「集計」において以下の工夫を凝らしています。

### 1. Self-Growing Dictionary (自己成長する型番辞書)
「型番入力の手間」と「表記ゆれ」は、正確なデータ管理の最大の障壁です。これを解決するため、**ユーザーの操作を通じて自動的に学習・成長する辞書機能**を実装しました。

- **仕組み (Mechanism):**
  ユーザーが製品を登録・更新する際、入力された「型番」と「製品情報（名称、メーカー、カテゴリ）」の組み合わせを、システムが内部辞書（`model_suggestions`）に自動的にフィードバック・蓄積します。
- **効果 (Benefit):**
  一人のユーザーが正確な情報を登録すれば、それは全ユーザーの資産となります。次回以降、同じ型番を入力するユーザーには詳細情報が自動補完（サジェスト）されるため、**「入力コストの最小化」と「データ品質の統一」**が同時に達成されます。

### 2. Noise Filtering for Accurate Statistics (統計ノイズの徹底排除)
単にデータを集めるだけでは、意味のある「実績値」は算出できません。本アプリでは、**製品本来の耐久性（実力）**を測るため、統計処理において厳格な除外ロジック（ノイズフィルタリング）を適用しています。

- **購入状態による除外:**
  前の持ち主の使用状況が不明な「中古品」や「再生品」は、寿命計算の母数から除外されます。「新品」から使用開始された製品のみを追跡することで、正確な製品寿命を算出します。
- **廃棄理由による除外:**
  「落下による破損」や「紛失」など、ユーザーの過失や外部要因による廃棄は、製品の品質とは無関係です。これらを集計から外し、「自然故障」によって寿命を迎えたケースのみを抽出することで、**カタログには載っていないリアルな「故障率」と「平均寿命」**を可視化します

### Architecture Decision: Why Livewire over Next.js?

**「インフラの選定」ではなく「コアロジック」に集中するための、バックエンド主導の選択**

- **比較検討 (The Consideration):**
  ネイティブアプリ並みのUXを実現するために、React (Next.js) も検討候補にあがりました。近年のNext.js (App Router) は強力ですが、認証・ORM・ジョブキューといった本格的なバックエンド機能を構築する場合、エコシステム内の最適解を調査・選定し、それらを自力で統合し続けるコストが発生します。

- **選定理由 (The Reason):**
  本アプリ「Model Type」の本質的な価値は、UIの派手さではなく、複雑なデータモデリングと統計処理（寿命計算・コスト算出）の正確性にあります。Laravelは、これら「分析プラットフォーム」に必要な機能が全て統合（Batteries Included）されており、堅牢性が保証されています。「ライブラリのパズル」に時間を割くことなく、Livewireを用いて堅牢なビジネスロジックを直接UIに接続できる点が、本プロジェクトにおいて最もROI（投資対効果）が高いと判断しました。

## Setup & Installation

このプロジェクトのセットアップには、以下のコマンドを実行してください。これには依存関係のインストール、環境設定、データベースのセットアップ、フロントエンドのビルドが含まれます。

> [!IMPORTANT]
> **現在の作業ブランチは `public_page` です！**
> 最新の状態を閲覧したい方はお手数ですがブランチの移動をお願いします。

```bash
composer setup
```

コマンドが完了したら、以下のコマンドでサーバーを起動できます。

```bash
php artisan serve
```
